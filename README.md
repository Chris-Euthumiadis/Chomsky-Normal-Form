# CFG → Chomsky Normal Form Converter

Ένα πρόγραμμα σε **C** που μετατρέπει μια Context-Free Grammar (CFG) στην **Κανονική Μορφή Chomsky (CNF)**. Διαβάζει τους κανόνες παραγωγής από αρχείο κειμένου και εξάγει την ισοδύναμη γραμματική σε CNF.

---

## Τι είναι η Κανονική Μορφή Chomsky (CNF)

Μια γραμματική είναι σε CNF όταν **κάθε κανόνας** έχει ακριβώς μία από τις παρακάτω μορφές:

```
A → BC      (δύο μη-τερματικά σύμβολα)
A → a       (ένα τερματικό σύμβολο)
S₀ → ε      (μόνο το αρχικό σύμβολο μπορεί να παράγει κενό)
```

Η CNF είναι θεμελιώδης στη θεωρία υπολογισμού και χρησιμοποιείται ευρέως σε αλγόριθμους ανάλυσης (π.χ. CYK algorithm).

---

## Παράδειγμα

**Είσοδος** (`eisodos.txt`):
```
S -> ASA | aB
A -> B | S
B -> b | e
```

**Έξοδος** (`eksodos.txt`):
```
S -> a | SA | AS | AU1 | U2B
A -> b | a | SA | AS | AU3 | U4B
B -> b
S0 -> a | SA | AS | AU5 | U6B
U1 -> SA
U2 -> a
U3 -> SA
U4 -> a
U5 -> SA
U6 -> a
```

---

## Αλγόριθμος — Βήματα Μετατροπής

Η μετατροπή γίνεται σε **4 βήματα** κατά σειρά:

### Βήμα 1 — Νέο αρχικό σύμβολο
Προστίθεται νέος κανόνας `S0 → S` ώστε το αρχικό σύμβολο να μην εμφανίζεται στα δεξιά κανενός κανόνα.

### Βήμα 2 — Απαλοιφή κανόνων ε (ε-productions)
Αφαιρούνται όλοι οι κανόνες της μορφής `A → ε` (εκτός από το `S0 → ε`). Για κάθε κανόνα που περιέχει στα δεξιά ένα μη-τερματικό `A` το οποίο μπορεί να παράγει `ε`, προστίθενται νέες παραλλαγές χωρίς το `A`.

### Βήμα 3 — Απαλοιφή unit productions
Αφαιρούνται κανόνες της μορφής `A → B` (ένα μόνο μη-τερματικό στα δεξιά). Αντικαθίστανται από τους κανόνες στους οποίους παράγει το `B`, αποφεύγοντας κυκλικές αντικαταστάσεις.

### Βήμα 4 — Κανονικοποίηση δεξιών μερών
Κάθε κανόνας με δεξί μέρος μήκους ≥ 3 ή με ανάμεικτα τερματικά/μη-τερματικά σπάει με χρήση βοηθητικών συμβόλων `U1, U2, ...`:
- Τερματικό `a` μέσα σε μακρύ δεξί → νέος κανόνας `Ui → a`
- Μακρύ δεξί → αλυσιδωτή αποσύνθεση σε ζεύγη: `A → U1U2`, `U2 → U3U4`, κ.ο.κ.

---

## Δομές Δεδομένων

Το πρόγραμμα χρησιμοποιεί **συνδεδεμένες λίστες** για να αναπαραστήσει τους κανόνες παραγωγής:

```
aristera[]   →  πίνακας συμβολοσειρών με τα αριστερά μέρη (π.χ. "S", "A", "U1")
head[]/tail[] →  ένας κόμβος head και tail ανά γραμμή (κανόνα)
struct dexia  →  κόμβος λίστας με ένα δεξί μέρος (str) και δείκτη next
```

Κάθε γραμμή κανόνα αντιστοιχεί σε έναν κόμβο-αρχή (`head[i]`) και οι εναλλακτικές παραγωγές συνδέονται ως λίστα.

Επιπλέον, δύο βοηθητικές λίστες (`svismena_e`, `svismenoi_kanones`) κρατούν ιστορικό των διαγραμμένων κανόνων ώστε να αποφεύγονται επαναλήψεις και κυκλικές αντικαταστάσεις.

## Μορφή Αρχείων

### Αρχείο εισόδου (`eisodos.txt`)

Κάθε γραμμή ορίζει έναν κανόνα παραγωγής:

```
<Μη-τερματικό> -> <παραγωγή1> | <παραγωγή2> | ...
```

Κανόνες:
- Τα **κεφαλαία γράμματα** (`S`, `A`, `B`, ...) είναι μη-τερματικά σύμβολα.
- Τα **μικρά γράμματα** (`a`, `b`, ...) είναι τερματικά σύμβολα.
- Το **`e`** συμβολίζει την κενή λέξη (ε).
- Το αρχικό σύμβολο είναι πάντα το **`S`**.
- Όλα τα σύμβολα πρέπει να ανήκουν στο αλφάβητο που ορίζεται μέσα στον κώδικα (`V[]`, `alphavito[]`).

### Αρχείο εξόδου (`eksodos.txt`)

Γράφεται αυτόματα από το πρόγραμμα στον ίδιο φάκελο, με την ισοδύναμη γραμματική σε CNF.

---

## Μεταγλώττιση & Εκτέλεση

### Απαιτήσεις
- Μεταγλωττιστής C (gcc / clang)
- Αρχείο `eisodos.txt` στον ίδιο κατάλογο με το εκτελέσιμο

### Μεταγλώττιση

```bash
gcc -o cnf main.c
```

### Εκτέλεση

```bash
./cnf
```

Το αποτέλεσμα αποθηκεύεται στο `eksodos.txt` και εκτυπώνεται και στην κονσόλα.
